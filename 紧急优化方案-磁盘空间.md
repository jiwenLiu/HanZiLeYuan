### 当前状态

### 方案2: 视频压缩（需要时间）

**使用 FFmpeg 压缩**:
```powershell
# 压缩所有视频到720p, 降低码率
foreach ($file in Get-ChildItem "D:\PC_test\Videos_Full_Backup\*.mp4") {
    $output = "D:\PC_test\Videos_Compressed\$($file.Name)"
    ffmpeg -i $file.FullName -vf scale=1280:720 -c:v libx264 -crf 28 -preset fast -c:a aac -b:a 96k $output
}
```

**预期效果**:
- 每个视频: 50MB → 15-20MB
- 总大小: 1.4GB → ~500MB
- 画质: 轻微下降，但可接受

3. **添加缓存机制**
```typescript
// 下载到 cacheDir，支持离线播放
private async downloadVideo(url: string, videoId: string): Promise<string> {
  const cacheDir = `${this.context.cacheDir}/videos`
  const cachePath = `${cacheDir}/${videoId}.mp4`
  
  if (fs.accessSync(cachePath)) {
    return cachePath  // 已缓存
  }
  
  // 下载视频
  await this.downloadFile(url, cachePath)
  return cachePath
}
```

**优势**:
- APK大小: 1.4GB → <50MB
- 按需下载: 节省存储空间
- 易于更新: 无需发版

**劣势**:
- 需要网络连接
- 首次播放需要下载
- 需要云存储服务

```typescript
interface VideoStorage {

  path: string
  size: number
  downloadable: boolean
}

private getVideoSource(video: VideoInfo): string {
  // 1. 检查本地是否存在
  const localPath = `${this.context.filesDir}/videos/${video.id}.mp4`
  if (fs.accessSync(localPath)) {
    return localPath
  }
  
  // 2. 检查缓存
  const cachePath = `${this.context.cacheDir}/videos/${video.id}.mp4`
  if (fs.accessSync(cachePath)) {
    return cachePath
  }
  
  // 3. 使用在线URL
  return video.url
}
```
