# 一键部署和运行指南

## 🚨 当前问题

你看到的错误：
```
⚠️ 视频文件不存在: 第1课《马牛羊》.mp4
```

**原因**: 视频文件还没有推送到设备的cacheDir目录

## ✅ 解决方案（2步）

### 步骤1：推送视频到设备

**双击运行这个脚本**:
```
deploy_videos_simple.bat
```

**这个脚本会**:
1. 检查 `hanzileyuan_videos` 文件夹中的48个视频
2. 连接到你的设备
3. 在设备上创建目录: `/data/storage/el2/base/haps/entry/cache/hanzileyuan_videos`
4. 逐个推送所有视频文件

**需要时间**: 10-15分钟（推送2GB视频）

**预期输出**:
```

✅ 设备已连接
✅ 目录创建完成

...
[100%] 推送: 第48课...

```

## ❓ 常见问题

### Q: 为什么不能直接用rawfile？
**A**: 
- rawfile会把视频打包在APK中，APK会变成2GB
- rawfile会一次性加载到内存，导致OOM（内存不足）
- 不支持懒加载

### Q: cacheDir方案的优势？
**A**:
- APK只有50MB（不包含视频）
- 视频在设备缓存目录，不占用应用空间
- 支持懒加载，内存占用低（~150MB）
- 安装成功率高

### Q: 推送需要多久？
**A**: 10-15分钟（48个视频，约2GB）

### Q: 系统会清理cacheDir吗？
**A**: 可能会。建议后续添加视频完整性检查功能。

### Q: 可以只推送部分视频吗？
**A**: 可以。修改 `deploy_videos_simple.bat` 脚本，只复制需要的视频。

## 🎯 工作原理

### 1. 视频存储
```
设备路径: /data/storage/el2/base/haps/entry/cache/hanzileyuan_videos/
├── 第1课《马牛羊》.mp4
├── 第2课《鸡狗猪猫》.mp4
└── ... (48个视频)
```

### 2. 应用访问
```typescript
// VideoFileService.ets
this.videoBasePath = `${this.context.cacheDir}/hanzileyuan_videos`

getVideoPath(videoFileName: string): string {
  return `${this.videoBasePath}/${videoFileName}`
}
```

### 3. Video组件
```typescript
// VideoPlayerPage.ets
Video({
  src: this.videoSrc,  // 直接使用文件路径
  controller: this.videoController
})
```

### 4. 懒加载
- Video组件只在播放时读取文件
- 不会一次性加载所有视频
- 支持流式播放

## 📊 性能对比

| 方案 | APK大小 | 内存占用 | 懒加载 | 安装时间 |
|------|---------|---------|--------|---------|
| rawfile | 2.0GB | 500MB+ | ❌ | 3-5分钟 |
| cacheDir | <50MB | ~150MB | ✅ | 10-20秒 |
