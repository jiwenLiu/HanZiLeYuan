# 构建测试说明

## 配置修正说明

### 问题
之前的配置中 `consoleDisable: true` 放在了错误的位置，导致构建失败。

### 解决方案
移除了 `consoleDisable` 配置，改为在混淆规则文件中使用 `-remove-log` 选项来移除console日志。

---

## 当前配置

### entry/build-profile.json5
```json5
"buildOptionSet": [
  {
    "name": "release",
    "arkOptions": {
      "obfuscation": {
        "ruleOptions": {
          "enable": true,
          "files": [
            "./obfuscation-rules.txt"
          ]
        }
      }
    }
  }
]
```

### entry/obfuscation-rules.txt
```txt
-enable-property-obfuscation
-enable-toplevel-obfuscation
-enable-filename-obfuscation
-enable-export-obfuscation
-compact
-remove-log  # 移除所有console日志

# 保留必要的全局名称
-keep-global-name
EntryAbility
EntryBackupAbility

# 保留数据模型的属性名
-keep-property-name
id
title
subtitle
emoji
duration
status
category
videoUrl
videoId
progress
lastPlayTime
currentTime
totalTime
stars
```

## 常见问题

### Q1: 构建失败怎么办？
**A**: 
1. 检查 `build-profile.json5` 语法是否正确
2. 确认 `obfuscation-rules.txt` 文件存在
3. 运行 `hvigorw clean` 清理后重试

### Q2: 混淆后应用崩溃？
**A**: 
检查是否有属性名被混淆，在 `obfuscation-rules.txt` 中添加 `-keep-property-name` 保留该属性。

### Q3: 如何关闭混淆？
**A**: 
将 `enable: true` 改为 `enable: false`，或在混淆规则文件开头添加 `-disable-obfuscation`。
