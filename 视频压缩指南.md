# 视频压缩指南

## 问题说明

HarmonyOS的Video组件对视频源有以下限制：
1. **rawfile方式**：适合小文件（<30MB），超过此大小可能无法播放
2. **本地文件路径**：Video组件可能不支持沙箱目录的绝对路径
3. **网络URL**：支持，但需要网络连接

当前27个视频文件大小在40-70MB之间，超过了rawfile的推荐大小。

## 解决方案：压缩视频文件

### 方法1：使用FFmpeg批量压缩（推荐）

```bash
# 安装FFmpeg（如果还没安装）
# Windows: 下载 https://ffmpeg.org/download.html
# Mac: brew install ffmpeg
# Linux: sudo apt install ffmpeg

# 批量压缩脚本（Windows PowerShell）
$videos = Get-ChildItem "entry\src\main\resources\rawfile\videos\*.mp4"
foreach ($video in $videos) {
    $output = "compressed_" + $video.Name
    ffmpeg -i $video.FullName -vcodec h264 -acodec aac -b:v 1.5M -b:a 128k $output
}

# 批量压缩脚本（Mac/Linux Bash）
for file in entry/src/main/resources/rawfile/videos/*.mp4; do
    filename=$(basename "$file")
    ffmpeg -i "$file" -vcodec h264 -acodec aac -b:v 1.5M -b:a 128k "compressed_$filename"
done
```

### 方法2：调整压缩参数

根据视频内容调整参数：

```bash
# 高质量（目标：25-30MB）
ffmpeg -i input.mp4 -vcodec h264 -acodec aac -b:v 2M -b:a 128k output.mp4

# 中等质量（目标：15-20MB）
ffmpeg -i input.mp4 -vcodec h264 -acodec aac -b:v 1.5M -b:a 128k output.mp4

# 低质量（目标：10-15MB）
ffmpeg -i input.mp4 -vcodec h264 -acodec aac -b:v 1M -b:a 96k output.mp4
```

### 方法3：降低分辨率

如果码率压缩还不够：

```bash
# 降低到720p
ffmpeg -i input.mp4 -vf scale=1280:720 -vcodec h264 -acodec aac -b:v 1.5M output.mp4

# 降低到480p
ffmpeg -i input.mp4 -vf scale=854:480 -vcodec h264 -acodec aac -b:v 1M output.mp4
```

## 压缩后的步骤

1. 将压缩后的视频替换原文件
2. 确认视频编码为H.264（视频）+ AAC（音频）
3. 重新编译应用
4. 测试播放

## 验证视频信息

```bash
# 查看视频编码信息
ffmpeg -i video.mp4

# 查看文件大小
ls -lh video.mp4  # Mac/Linux
Get-Item video.mp4 | Select-Object Name, Length  # Windows PowerShell
```

## 预期效果

- 视频文件：40-70MB → 15-25MB
- APK大小：1.3GB → 约500-700MB
- 播放：正常，自动播放功能正常
