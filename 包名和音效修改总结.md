## 三、代码修改

### 1. 新增音效服务
创建了 `entry/src/main/ets/services/SoundEffectService.ets`

**功能特性**:
- 支持背景音乐循环播放
- 支持点击音效播放
- 背景音乐和音效可同时播放（使用两个独立的AVPlayer）
- 自动资源管理和释放

**主要方法**:
- `initialize(context)` - 初始化服务
- `playBackgroundMusic(audioPath)` - 播放背景音乐（循环）
- `stopBackgroundMusic()` - 停止背景音乐
- `playClickSound()` - 播放点击音效
- `release()` - 释放所有资源

## 四、音效使用说明

### 在新页面中添加音效支持

1. **导入音效服务**:
```typescript
import { soundEffectService } from '../services/SoundEffectService'
```

2. **在onClick事件中添加音效**:
```typescript
.onClick(() => {
  soundEffectService.playClickSound();
  // 你的点击处理逻辑
})
```

3. **播放背景音乐（仅首页）**:
```typescript
async aboutToAppear() {
  const context = getContext(this) as common.UIAbilityContext;
  await soundEffectService.initialize(context);
  await soundEffectService.playBackgroundMusic('audios/muzhiqin.mp3');
}

async aboutToDisappear() {
  await soundEffectService.stopBackgroundMusic();
}
```

## 八、技术说明

### AVPlayer使用
- 使用HarmonyOS的 `media.AVPlayer` API
- 通过 `fdSrc` 设置音频源（文件描述符）
- 背景音乐设置 `loop = true` 实现循环播放
- 音效播放完成后自动释放资源

### 资源管理
- 音频文件存放在 `rawfile/audios/` 目录
- 使用 `resourceManager.getRawFd()` 获取文件描述符
- 自动管理AVPlayer的生命周期

### 性能优化
- 背景音乐使用单例模式，全局共享
- 音效播放器在播放完成后自动释放
- 避免音效重叠播放
