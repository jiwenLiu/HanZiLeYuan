# 外部视频存储方案说明

## 方案概述

将所有视频文件从APK内部（rawfile）移动到设备外部存储（`/storage/emulated/0/hanzileyuan_videos`），大幅减小APK体积。

## 优势

### 1. APK体积大幅减小
- **优化前**: 1.4GB（包含29个视频）
- **优化后**: <50MB（不包含视频）
- **减小**: 约1.35GB（96%）

### 2. 解决安装问题
- ✅ 不再出现"insufficient disk memory"错误
- ✅ 安装速度更快
- ✅ 支持更多设备

### 3. 灵活的视频管理
- ✅ 可以随时添加/删除视频
- ✅ 无需重新发版
- ✅ 用户可以自行管理视频文件

### 4. 内存优化
- ✅ 不使用rawfile，避免一次性加载到内存
- ✅ 流式读取，内存占用更低
- ✅ 支持更大的视频文件

## 实施步骤

### 步骤1：移动视频文件到本地目录

运行脚本：
```powershell
.\move_videos_to_external.bat
```

**执行结果**:
- 将 `entry\src\main\resources\rawfile\videos\*.mp4` 移动到 `hanzileyuan_videos\`
- 统计文件数量和大小
- 显示目标目录路径

### 步骤2：部署视频文件到设备

运行脚本：
```powershell
.\deploy_videos_to_device.bat
```

**执行结果**:
- 检查设备连接
- 在设备上创建 `/storage/emulated/0/hanzileyuan_videos` 目录
- 推送所有视频文件到设备
- 验证文件完整性

**注意**: 推送29个视频（约1.4GB）需要5-10分钟，请耐心等待。

### 步骤3：重新编译应用

```powershell
# 清理构建
hvigorw clean

# 重新编译
hvigorw assembleHap
```

**预期结果**:
- APK大小从1.4GB降到<50MB
- 编译时间大幅缩短
- 安装速度更快

### 步骤4：安装并测试

1. 在DevEco Studio中点击"Run"
2. 应用安装成功
3. 打开应用，进入"看视频"页面
4. 点击任意视频，验证播放功能

## 代码修改说明

### 1. 新增VideoFileService服务

**文件**: `entry/src/main/ets/services/VideoFileService.ets`

**功能**:
- 管理外部视频文件路径
- 检查视频文件是否存在
- 获取视频文件信息
- 列出所有可用视频

**关键方法**:
```typescript
// 获取视频路径
getVideoPath(videoFileName: string): string

// 检查视频是否存在
checkVideoExists(videoFileName: string): boolean

// 列出所有视频
listAvailableVideos(): string[]
```

### 2. 修改VideoPlayerPage

**文件**: `entry/src/main/ets/pages/VideoPlayerPage.ets`

**主要修改**:

1. **导入VideoFileService**
```typescript
import { VideoFileService } from '../services/VideoFileService'
```

2. **初始化服务**
```typescript
private videoFileService: VideoFileService | null = null

async aboutToAppear() {
  const context = getContext(this) as common.UIAbilityContext
  this.videoFileService = VideoFileService.getInstance(context)
  // ...
}
```

3. **修改loadVideo方法**
```typescript
loadVideo() {
  const video = this.videoList[this.currentVideoIndex]
  const videoFileName = `第${video.id}课${video.subtitle}.mp4`
  
  // 使用外部路径
  this.videoSrc = this.videoFileService!.getVideoPath(videoFileName)
}
```

4. **修改Video组件**
```typescript
// 修改前
Video({
  src: $rawfile(this.videoSrc),  // ❌ 使用rawfile
  controller: this.videoController
})

// 修改后
Video({
  src: this.videoSrc,  // ✅ 直接使用文件路径
  controller: this.videoController
})
```

## 使用流程

### 开发阶段
1. 运行 `move_videos_to_external.bat` - 移动视频到本地
2. 运行 `deploy_videos_to_device.bat` - 部署到设备
3. 编译安装应用
4. 测试视频播放

### 发布阶段
1. 编译APK（不包含视频，<50MB）
2. 准备视频文件包（hanzileyuan_videos.zip）
3. 提供安装说明：
   - 安装APK
   - 解压视频包到 `/storage/emulated/0/hanzileyuan_videos`
   - 启动应用

### 用户使用
1. 安装应用
2. 首次启动时，应用会检查视频目录
3. 如果视频不存在，显示提示信息
4. 用户手动复制视频文件到指定目录
5. 重启应用，正常使用

## 错误处理

### 1. 视频目录不存在
**错误信息**: "外部视频目录不存在"
**解决方案**: 
- 运行 `deploy_videos_to_device.bat`
- 或手动创建目录并复制视频

### 2. 视频文件不存在
**错误信息**: "视频文件不存在: 第X课..."
**解决方案**:
- 检查文件名是否正确
- 确认视频文件已复制到设备
- 检查文件权限

### 3. 权限被拒绝
**错误信息**: "Permission denied"
**解决方案**:
- 在应用设置中授予存储权限
- 重启应用

## 注意事项

### 1. 首次部署
- 视频推送需要5-10分钟
- 确保USB连接稳定
- 确保设备有足够存储空间（至少2GB）

### 2. 权限管理
- 首次启动时会请求存储权限
- 用户必须授予权限才能播放视频
- 如果权限被拒绝，需要在设置中手动授予

### 3. 文件管理
- 用户可以自行管理视频文件
- 删除视频文件不会影响应用运行
- 可以添加新的视频文件（需要更新代码中的视频列表）

### 4. 版本更新
- 应用更新不会影响视频文件
- 视频文件可以独立更新
- 建议提供视频文件的在线下载功能
