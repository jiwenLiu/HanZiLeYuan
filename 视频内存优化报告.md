# 视频内存优化报告

## 问题诊断

### 1. ❌ 原问题：autoPlay=true
**位置**: `VideoPlayerPage.ets:292`
```typescript
.autoPlay(true)  // ❌ 会立即加载视频到内存
```

**影响**:
- 页面加载时立即播放，瞬间占用大量内存
- 多个视频切换时会导致内存峰值
- 可能触发 OOM (Out of Memory)

### 2. ⚠️ 当前问题：使用 rawfile 存储大视频
**位置**: `VideoPlayerPage.ets:286`
```typescript
src: $rawfile(this.videoSrc)  // ⚠️ rawfile会一次性加载到内存
```

**影响**:
- rawfile 读取时会一次性加载整个文件到内存
- 29个视频文件，每个约10-20MB
- 总大小约300-500MB，必然触发 OOM

## 已完成优化

### ✅ 1. 移除 autoPlay
**修改**: `VideoPlayerPage.ets:292`
```typescript
.autoPlay(false)  // ✅ 改为手动播放
```

**效果**:
- 页面加载时不会立即播放
- 用户点击播放按钮才开始加载视频
- 减少内存峰值

## 待优化项（重要）

### 🔴 优先级1：视频文件迁移到 filesDir

**当前方案（有问题）**:
```
entry/src/main/resources/rawfile/videos/*.mp4  ❌
```

**推荐方案**:
```
context.filesDir/videos/*.mp4  ✅
```

**实施步骤**:

1. **创建视频复制服务**
```typescript
// VideoFileService.ets
import { common } from '@kit.AbilityKit'
import fs from '@ohos.file.fs'

export class VideoFileService {
  private static instance: VideoFileService
  private context: common.UIAbilityContext
  
  static getInstance(context: common.UIAbilityContext): VideoFileService {
    if (!VideoFileService.instance) {
      VideoFileService.instance = new VideoFileService(context)
    }
    return VideoFileService.instance
  }
  
  private constructor(context: common.UIAbilityContext) {
    this.context = context
  }
  
  // 首次启动时，将rawfile视频复制到filesDir
  async copyVideosToFilesDir(): Promise<void> {
    const videosDir = `${this.context.filesDir}/videos`
    
    // 检查目录是否存在
    if (!fs.accessSync(videosDir)) {
      fs.mkdirSync(videosDir)
    }
    
    // 复制视频文件（仅首次或更新时）
    const videoList = [
      '第1课《马牛羊》.mp4',
      '第2课《鸡狗猪猫》.mp4',
      // ... 其他视频
    ]
    
    for (const video of videoList) {
      const destPath = `${videosDir}/${video}`
      if (!fs.accessSync(destPath)) {
        // 从rawfile复制到filesDir
        await this.copyFromRawfile(`videos/${video}`, destPath)
      }
    }
  }
  
  private async copyFromRawfile(rawfilePath: string, destPath: string): Promise<void> {
    const resourceMgr = this.context.resourceManager
    const fileData = await resourceMgr.getRawFileContent(rawfilePath)
    const file = fs.openSync(destPath, fs.OpenMode.CREATE | fs.OpenMode.WRITE_ONLY)
    fs.writeSync(file.fd, fileData.buffer)
    fs.closeSync(file)
  }
  
  getVideoPath(videoName: string): string {
    return `${this.context.filesDir}/videos/${videoName}`
  }
}
```

### 🟡 优先级2：LazyForEach 优化（WatchPage）

**当前**: 使用 ForEach 渲染所有29个视频卡片
**问题**: 一次性创建所有组件，占用内存

**优化方案**:
```typescript
// WatchPage.ets
class VideoDataSource implements IDataSource {
  private videos: VideoInfo[] = []
  
  constructor(videos: VideoInfo[]) {
    this.videos = videos
  }
  
  totalCount(): number {
    return this.videos.length
  }
  
  getData(index: number): VideoInfo {
    return this.videos[index]
  }
  
  registerDataChangeListener(listener: DataChangeListener): void {}
  unregisterDataChangeListener(listener: DataChangeListener): void {}
}

// 使用 LazyForEach
@State videoDataSource: VideoDataSource = new VideoDataSource(this.allVideos)

LazyForEach(this.videoDataSource, (video: VideoInfo, index: number) => {
  // 视频卡片组件
  this.VideoCard(video, index)
}, (video: VideoInfo) => video.id)
```

## 注意事项

1. **首次启动时间**: 从 rawfile 复制到 filesDir 需要时间，建议：
   - 显示加载进度
   - 后台异步复制
   - 使用增量更新（仅复制新增/更新的视频）

2. **存储空间**: filesDir 会占用用户存储空间
   - 29个视频约 300-500MB
   - 需要检查可用空间
   - 提供清理缓存功能

3. **版本更新**: 视频更新时需要重新复制
   - 使用版本号标记
   - 检测文件MD5
   - 增量更新机制
