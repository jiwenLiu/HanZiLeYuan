# 签名验证失败解决方案

## 错误信息
```
error: signature verification failed due to not trusted app source
code: 9568322
```

## 原因分析
设备不信任应用的签名来源，这通常发生在：
1. 使用release签名但设备未授权
2. 设备安全设置限制
3. 签名证书过期或无效

## 解决方案

### ✅ 方案1：切换到Debug签名（已执行）

**修改**: `build-profile.json5`
```json5
{
  "name": "default",
  "signingConfig": "default",  // ✅ 使用debug签名，而非release
}
```

**执行步骤**:
```powershell
# 1. 清理构建
hvigorw clean

# 2. 重新编译
hvigorw assembleHap

# 3. 安装
# DevEco Studio会自动安装
```

### 方案2：在设备上启用开发者选项

#### 真机设置
1. **启用开发者模式**
   - 打开"设置" → "关于手机"
   - 连续点击"版本号"7次
   - 提示"您已处于开发者模式"

2. **启用USB调试**
   - 返回"设置" → "系统和更新" → "开发者选项"
   - 启用以下选项：
     - ✅ USB调试
     - ✅ 安装未知来源应用
     - ✅ 仅充电模式下允许ADB调试
     - ✅ 允许通过USB安装应用

3. **信任电脑**
   - 连接USB后，设备会弹出"是否允许USB调试"
   - 点击"确定"并勾选"始终允许"

#### 模拟器设置
1. 打开DevEco Studio的Device Manager
2. 选择你的模拟器
3. 点击设置图标（齿轮）
4. 确认以下设置：
   - Developer Mode: ✅ Enabled
   - USB Debugging: ✅ Enabled

### 方案3：重新生成签名证书

如果debug证书过期或损坏：

1. **删除旧证书**
```powershell
Remove-Item "D:\cert\debug.p12" -Force
Remove-Item "D:\cert\debug.cer" -Force
Remove-Item "D:\cert\汉字乐园DebugDebug.p7b" -Force
```

2. **在DevEco Studio中重新生成**
   - File → Project Structure
   - Project → Signing Configs
   - 点击"Automatically generate signature"
   - 选择"Debug"模式
   - 点击"OK"

### 方案4：使用命令行手动安装

如果DevEco Studio安装失败，可以手动安装：

```powershell
# 1. 查找设备
hdc list targets

# 2. 卸载旧版本
hdc shell bm uninstall -n com.chinasoft.tools.hanzileyuan

# 3. 手动安装
hdc install entry\build\default\outputs\default\entry-default-signed.hap

# 4. 启动应用
hdc shell aa start -a EntryAbility -b com.chinasoft.tools.hanzileyuan
```

### 方案5：检查设备存储空间

虽然错误是签名问题，但也可能是存储空间不足导致的：

```powershell
# 检查设备存储
hdc shell df -h

# 清理缓存
hdc shell pm clear com.chinasoft.tools.hanzileyuan
```

## 推荐执行顺序

### 第1步：使用Debug签名（已完成）
✅ 已修改 `build-profile.json5`，使用debug签名

### 第2步：清理并重新编译
```powershell
hvigorw clean
hvigorw assembleHap
```

### 第3步：检查设备设置
- 确认USB调试已启用
- 确认允许安装未知来源应用
- 确认电脑已被信任

### 第4步：如果还是失败
- 尝试重启设备
- 尝试重启DevEco Studio
- 尝试更换USB端口
- 尝试使用命令行手动安装

## 常见问题

### Q1: 找不到"开发者选项"
**A**: 需要先启用开发者模式（连续点击版本号7次）

### Q2: USB调试已启用但还是失败
**A**: 
1. 检查USB连接模式（应该是"传输文件"而非"仅充电"）
2. 重新插拔USB线
3. 在设备上撤销USB调试授权，重新授权

### Q3: 模拟器无法安装
**A**:
1. 重启模拟器
2. 在Device Manager中"Wipe Data"（清除数据）
3. 重新创建模拟器

### Q4: 证书过期
**A**: 
1. 删除旧证书
2. 在DevEco Studio中重新生成
3. 重新编译

## 验证步骤

安装成功后，应该看到：
```
Install Success
```

如果看到以下信息，说明安装成功：
```
$ hdc shell bm install -p data/local/tmp/xxx
Install Success
```
